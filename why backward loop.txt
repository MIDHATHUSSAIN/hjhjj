🔁 Why backward loop is used here?

You wrote:

for (int i = _connectedClients.Count - 1; i >= 0; i--)

This is called a backward iteration.

👉 The reason is:
You are removing items from the list while looping:

_connectedClients.RemoveAt(i);
⚠️ What goes wrong with forward loop?

If you do a normal loop:

for (int i = 0; i < _connectedClients.Count; i++)
❌ Problem: Index shifting

Example:

List = [A, B, C, D]
Index   0  1  2  3

Now suppose you remove B at index 1.

After removal:

List = [A, C, D]
Index   0  1  2

But your loop does i++ → i = 2

👉 Now you skip C (which moved to index 1)

So:

Some clients won’t be checked
Some dead clients may remain in list
Bugs become inconsistent and hard to trace
✅ Why backward loop fixes this

Backward loop:

i = Count - 1 → 3 → 2 → 1 → 0

If you remove item:

It only affects elements after it
But you already processed those

So no skipping happens ✔️
